# Phase Summaries

โฟลเดอร์นี้เก็บสรุปงานท้ายแต่ละ phase ไว้ข้าง `README.md` ที่ root ของ repo — ใช้ทบทวนตอนปิด phase / ก่อนเริ่ม phase ถัดไป

## กฎสั้นๆ

- สร้างไฟล์ markdown **หนึ่งไฟล์ต่อ phase** ที่ปิดหรือทำบางส่วนแล้ว เช่น `phase-g.md`, `phase-f.md`
- ภาษาไทยเป็นหลัก
- แต่ละไฟล์มีหัวข้ออย่างน้อย:
  - **สิ่งที่ทำแล้ว**
  - **งานคงเหลือ**
  - **เกตที่ค้าง**
  - **commits ที่เกี่ยวข้อง**
  - **วันที่**
- ชี้กลับไปยัง phase doc / gate artifacts ใต้ `knowledge/05-architecture/phases/` เมื่อมี
- อย่า invent งานที่ยังไม่ทำ — ถ้าไม่แน่ใจ ให้เขียนเฉพาะที่พิสูจน์ได้จาก phase doc หรือ commit
